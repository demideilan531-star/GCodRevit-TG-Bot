# Регламент публикации отчётов Gmail в Telegram

## 1. Значение команды «Сделай пост»

В рамках проекта **GCodRevit-TG-Bot** фраза **«Сделай пост»** всегда означает одну законченную операцию:

1. Получить актуальные данные Gmail.
2. Составить фактический отчёт на русском языке.
3. Создать новую инфографику по утверждённому шаблону.
4. Опубликовать **один Telegram-пост**, в котором фотография и текст находятся вместе.
5. Проверить, что Telegram действительно принял публикацию.
6. Только после подтверждения сообщить пользователю, что пост опубликован.

Команда не означает «показать черновик», «подготовить текст» или «создать картинку без публикации», если пользователь отдельно не уточнил обратное.

## 2. Источник данных

Основной источник — Gmail подключённого пользователя.

Перед публикацией нужно:

- проверить письма за последние 65 минут, если пользователь не указал другой период;
- исключить Спам и Корзину;
- прочитать содержимое новых писем, а не судить только по теме;
- не придумывать отправителей, темы, суммы, даты, статусы или требуемые действия;
- использовать московское время, если пользователь не попросил другой часовой пояс.

Одноразовые письма, не требующие действий, нужно помечать ярлыком **«Одноразовые письма»** и убирать из входящих. К ним относятся ожидаемые коды, обычные подтверждения входа, сервисные уведомления, чеки и подтверждения покупок. Подозрительные входы, спорные операции и угрозы безопасности одноразовыми не считать.

## 3. Обязательная структура отчёта

Каждый отчёт должен содержать:

- дату и точное время проверки;
- проверенный период;
- общее количество новых писем;
- количество срочных или технических писем;
- количество ответов по работе или учёбе;
- количество приглашений;
- количество финансовых вопросов;
- количество угроз безопасности;
- количество одноразовых писем;
- краткий вывод;
- конкретное действие, если оно требуется.

Для каждого важного письма указываются:

- отправитель;
- тема;
- краткая суть;
- требуемое действие.

Если новых писем нет, обязательно использовать формулировку:

> За последний час новых писем нет.

## 4. Утверждённый дизайн изображения

Изображение создаётся в вертикальном формате примерно **4:5**. Рекомендуемый размер — 1080×1350 или 1200×1500 пикселей.

### Визуальный стиль

- тёмно-синий или почти чёрный фон;
- ночной городской вид и рабочий стол с ноутбуком как фоновая композиция;
- крупный белый заголовок **«ОТЧЁТ ПО GMAIL»**;
- логотип Gmail слева;
- бейдж **GMAIL MONITOR BOT** справа;
- крупные карточки с закруглёнными углами;
- большие, легко читаемые цифры;
- минимальное количество мелкого текста;
- аккуратные цветные иконки категорий.

### Цвета категорий

- красный — срочные и технические письма;
- синий — работа и учёба;
- фиолетовый — приглашения;
- жёлтый — финансовые вопросы;
- зелёный — безопасность в норме;
- голубой — одноразовые письма и информационные показатели.

### Состав изображения

1. Заголовок и дата проверки.
2. Карточка периода анализа.
3. Крупная карточка количества новых писем.
4. Карточка главного вывода.
5. Отдельные строки категорий:
   - срочные письма;
   - работа/учёба;
   - приглашения;
   - финансы;
   - безопасность.
6. Отдельный блок одноразовых писем.
7. Короткий итог в нижней части.

Если важных событий больше трёх, на картинке показывается общий вывод и максимум три ключевых события. Остальные объединяются в строку «Ещё N писем». Это нужно, чтобы текст оставался крупным и читабельным.

На изображении запрещено заранее писать **«пост отправлен»** или **«доставка успешна»**. Такой статус допустим только после фактической публикации и проверки результата.

## 5. Формат текста под фотографией

Фотография и текст должны отправляться **одним сообщением Telegram** через метод `sendPhoto`, где текст используется как подпись к изображению.

Базовый шаблон подписи:

```text
📬 Отчёт по Gmail

🕒 Проверка: [дата и время]
⏱ Период: [начало] — [конец]

📨 Новых писем: [N]
⚠️ Требуют внимания: [N]
🗂 Одноразовых перемещено: [N]

Главное:
[главный вывод]

Что сделать:
[конкретное действие или «Действий не требуется»]

🚨 Срочные/технические: [N]
💼 Работа/учёба: [N]
📅 Приглашения: [N]
💳 Финансы: [N]
🔐 Безопасность: [N]
```

Если новых писем нет:

```text
📬 Отчёт по Gmail

🕒 Проверка: [дата и время]
⏱ Период: [начало] — [конец]

📨 Новых писем: 0
✅ За последний час новых писем нет.

🚨 Срочные/технические: 0
💼 Работа/учёба: 0
📅 Приглашения: 0
💳 Финансы: 0
🔐 Угрозы безопасности: 0
🗂 Одноразовых писем: 0
```

Текст должен быть компактным и помещаться в подпись к фотографии. Если данных слишком много, детали сокращаются до главного; пост не разделяется на фотографию и отдельное текстовое сообщение, если пользователь явно не попросил об этом.

## 6. Техническая последовательность публикации

1. Сформировать окончательный текст отчёта.
2. Записать тот же текст в `reports/latest.txt` в ветке `main`.
3. Создать изображение на основе тех же данных.
4. Запустить GitHub Actions workflow публикации.
5. Workflow должен отправить фотографию и подпись одним запросом Telegram `sendPhoto`.
6. Проверить шаг отправки в Telegram.
7. Только если шаг завершился успешно, считать пост опубликованным.

Коммит в GitHub сам по себе не является подтверждением публикации. Запуск workflow сам по себе также не является подтверждением публикации.

## 7. Проверка качества перед отправкой

Перед публикацией нужно проверить:

- дата и год указаны правильно;
- время и период не противоречат друг другу;
- количество писем совпадает со списком;
- категории посчитаны правильно;
- на картинке нет опечаток и обрезанного текста;
- нет вымышленных фактов;
- нет токенов, паролей, кодов подтверждения и других секретов;
- текст подписи соответствует картинке;
- действие сформулировано понятно;
- публикация представляет собой один пост: фото плюс текст.

## 8. Подтверждение результата

После успешной проверки использовать точную формулировку:

> Пост с фотографией и текстом опубликован в Telegram.

Если создан только коммит или запуск ещё выполняется:

> Публикация запущена, но доставка в Telegram ещё не подтверждена.

Если отправка не удалась:

> Пост не опубликован. Ошибка: [точная причина].

Запрещено утверждать, что пост отправлен, если шаг Telegram не был проверен.

## 9. Поведение по команде пользователя

Когда пользователь пишет **«Сделай пост»**:

- не задавать лишних уточняющих вопросов, если доступен актуальный отчёт;
- если актуального отчёта нет, сначала проверить Gmail за последние 65 минут;
- создать свежую картинку, а не повторно использовать устаревшую;
- использовать утверждённый визуальный шаблон;
- отправить картинку и текст одним Telegram-постом;
- проверить успешность публикации;
- сообщить пользователю фактический результат.

Этот регламент является основным правилом для всех будущих почтовых постов проекта.
