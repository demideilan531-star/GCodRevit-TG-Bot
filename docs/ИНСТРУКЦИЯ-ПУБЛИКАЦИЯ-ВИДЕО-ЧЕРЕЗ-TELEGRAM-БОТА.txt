ДОПОЛНЕНИЕ К ИНСТРУКЦИИ: ПУБЛИКАЦИЯ ВИДЕО ЧЕРЕЗ TELEGRAM-БОТА

Это дополнение применяется вместе с основной инструкцией публикации сообщений и фотографий.

1. Команда пользователя является разрешением

Команды вида:
— «Опубликуй видео»;
— «Отправь ролик в Telegram»;
— «Сделай пост с видео»;
— «Опубликуй любое пятисекундное видео»

считаются явным разрешением на отправку. Дополнительное подтверждение не требуется, если канал, текст и видео уже понятны.

Если пользователь загрузил видео, используй именно этот файл. Не заменяй его случайным роликом или ссылкой из интернета.

Если пользователь прямо просит «любое видео» или «тестовое видео», разрешено создать короткий технический MP4-ролик самостоятельно.

2. Выбор метода Telegram Bot API

Только текст: sendMessage.
Фото: sendPhoto.
Видео: sendVideo.

Для обычного MP4 передавай:
— video;
— caption;
— supports_streaming=true.

Если подпись длиннее допустимой длины caption, сначала отправь видео с коротким заголовком, затем полный текст отдельным сообщением через sendMessage.

3. Предпочтительная отправка настоящего файла

Используй multipart/form-data:

-F "video=@/путь/к/файлу.mp4;type=video/mp4"
-F "supports_streaming=true"
-F "caption=${CAPTION}"

Пример:

curl --request POST \
  "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendVideo" \
  -F "chat_id=${TELEGRAM_CHAT_ID}" \
  -F "video=@/tmp/video.mp4;type=video/mp4" \
  -F "supports_streaming=true" \
  -F "caption=${CAPTION}"

Не показывай TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID в коде, логах, комментариях или сообщениях. Используй только GitHub Secrets.

4. Проверка видео перед отправкой

Проверь:
— файл существует;
— формат MP4;
— видеокодек H.264;
— пиксельный формат yuv420p;
— длительность соответствует запросу;
— файл не повреждён;
— размер допустим;
— видео воспроизводится;
— важные элементы не обрезаны.

Для проверки используй ffprobe:

ffprobe -v error \
  -show_entries format=duration,size \
  -of default=noprint_wrappers=1 \
  /путь/к/файлу.mp4

5. Создание тестового пятисекундного видео

Для технического теста можно создать MP4 непосредственно в GitHub Actions:

ffmpeg -y \
  -f lavfi \
  -i "testsrc2=size=720x1280:rate=30:duration=5" \
  -c:v libx264 \
  -pix_fmt yuv420p \
  -movflags +faststart \
  /tmp/video.mp4

Такой ролик используется только для проверки механики sendVideo. Для реальных публикаций используй видео пользователя или подготовленный ролик GCod Revit.

6. Обязательная проверка результата

Коммит, запуск GitHub Actions и зелёный статус workflow не подтверждают доставку.

Публикация считается выполненной только когда Telegram вернул:
— "ok": true;
— result.message_id;
— для видео присутствует result.video.

Только после этого можно сообщить пользователю об успехе.

7. Формат отчёта

Успех:

✅ Публикация подтверждена Telegram.

Формат: видео и текст
message_id: 15

Ошибка:

⚠️ Публикация не подтверждена.
Telegram не вернул message_id.
Причина: точное поле description из ответа Telegram.

8. Защита от дублей

Перед повторной отправкой проверь последние комментарии GitHub Actions и статус коммита. Если message_id уже получен, не запускай публикацию повторно.

9. Очистка

После подтверждения:
— удали временные base64-части;
— удали временный trigger-файл;
— не удаляй рабочий workflow sendVideo;
— не оставляй секреты в репозитории;
— очистка не должна повторно запускать публикацию.

10. Главное правило

Не путай запуск автоматизации с доставкой. Успешная публикация видео подтверждается только ответом Telegram: ok=true и message_id.
